Operational Analytics

A single executive view of daily operations — throughput, labor utilization, and SLA adherence — so leaders can see the state of the operation at a glance.

Data & privacy

Originally built on a live operational database; rebuilt for this portfolio on fully synthetic data. Column names are preserved so the visuals and DAX rebind unchanged, but every value is regenerated — no real, nudged, or proprietary figures appear.

What it shows

  • Outbound utilization by hour and by facility
  • Scheduled vs. temp labor hours against actual throughput
  • Pick and checkout productivity (packs per active hour)
  • Trip-building load and bags per hour

How it’s built

Data model

Star schema: an operations fact table with date, shift, department, and location dimensions — structured so each KPI traces back to a single grain.

DAX patterns

Throughput per hour, utilization ratios, and SLA-adherence percentages, plus period-over-period comparisons handled with time-intelligence measures.

Refresh

Re-pointed from the production database to a synthetic Excel source, then a full refresh with confirmation that no original rows remain. Scheduled refresh keeps the demo current.
